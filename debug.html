LLDB Quick Usage Guide

Step 1: Compile Your Program with Debug Symbols

Before you can use LLDB to debug your program, you need to compile it with debug information included. This is done using the -g flag with clang: 

clang -g /Users/harzinn/dev/cs50/factorial.c -o /Users/harzinn/dev/cs50/factorial -lcs50

Step 2: Launch LLDB with Your Program

Start LLDB and load your program: 

lldb /Users/harzinn/dev/cs50/factorial

This will start LLDB and load the executable you want to debug.

Step 3: Set `DYLD_LIBRARY_PATH`

Set the `DYLD_LIBRARY_PATH` environment variable to include the directory where the CS50 library is located: 

(lldb) settings set target.env-vars DYLD_LIBRARY_PATH=/usr/local/lib

Step 4: Set Breakpoints

Set breakpoints at lines of interest in your source code. For example, to set a breakpoint at line 10: 

breakpoint set --file factorial.c --line 10

You can also set breakpoints at specific functions:

breakpoint set --name main

Step 5: Run the Program

Start running your program within LLDB: 

run

The program will run until it hits a breakpoint, at which point LLDB will return control to you.

Step 6: Step Through the Code

Once your program hits a breakpoint, you can step through your code line by line:

  • step or s: Step into the next line of code (including into functions).
  • next or n: Step over to the next line of code (skipping over function calls).
  • finish: Run until the current function returns.

Step 7: Inspect Variables

To inspect the value of variables, use the print command: 

print variable_name

For example:

print highest_votes

You can also watch for changes to a variable:

watch set variable variable_name

Step 8: Continue Execution

To continue running the program until the next breakpoint or the end of the program: 

continue

Step 9: Exit LLDB

To exit LLDB: 

quit

By following these steps, you can effectively debug your C program, set breakpoints, inspect variables, and watch how they change as your program executes. Happy debugging!
